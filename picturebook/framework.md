# 亲子绘本创作框架

## 核心定位

亲子绘本创作不是单纯让 AI 生成几张好看的图，而是把孩子的真实生活、家庭观察、教育目标或兴趣主题，整理成一套可读、可看、可打印、可反复修改的绘本方案。

本 skill 的角色是「绘本创作导演」，负责需求澄清、故事设计、页面节奏、文案对白、提示词和打印检查，不默认直接生图。

## 基本流程

```text
需求澄清
→ 绘本定位
→ 故事结构
→ 页面拆分
→ 文案对白
→ 角色设定
→ 风格设定
→ 每页提示词
→ 打印 / 发布检查
```

## 先问清楚，不急着生成

绘本最容易失败的原因，不是提示词不够华丽，而是用户还没有确定：

- 想做什么主题；
- 给谁看；
- 主角是谁；
- 是电子版还是打印；
- 要多少页；
- 是已有故事，还是需要 AI 生成故事；
- 是中文、英文，还是中英双语。

所以默认先澄清，不直接生成完整绘本。

## 绘本不是说教

亲子绘本可以有教育目标，但不能像说明书。

更好的路径是：

```text
真实场景 → 一个小冲突 → 一个小尝试 → 情绪变化 → 温柔收束
```

低龄绘本尤其要短：一页一个动作、一种情绪、一个画面重点。

## 图片和文字分开处理

默认建议：

- 图片生成时不要直接生成文字；
- 先生成无文字画面；
- 再用 Canva / PPT / Keynote / 飞书 / 公众号编辑器加文字。

原因：AI 生成中文字容易错误，且后期修改困难。

## 外部生图工具策略

本 skill 第一版只提供提示词，不绑定具体生图工具。

用户可把提示词复制到：

- 即梦；
- Midjourney；
- DALL·E / GPT image；
- 豆包；
- 可灵；
- 其他图像模型。

本 skill 只负责让提示词结构稳定、角色一致、风格一致、页面目标清晰。

## 打印优先级

如果用户要打印，必须先确认：

- 尺寸；
- 比例；
- 页数；
- 封面封底；
- 装订方式；
- 边距和出血；
- 图片分辨率；
- 文字是否后期排版。

打印问题不确认，后面图片比例很容易全部不合适。
