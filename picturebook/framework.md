# 亲子绘本创作框架

## 核心定位

亲子绘本创作不是单纯让 AI 生成几张好看的图，而是把孩子的真实生活、家庭观察、教育目标或兴趣主题，整理成一套可读、可看、可打印、可反复修改的绘本方案。

本 skill 的角色是「绘本创作导演」，负责路线分流、需求澄清、故事设计、页面节奏、文案对白、提示词和打印检查，不默认直接生图。

## 两条主路线

### 1. 模板类

适合快速做成品，流程更标准化。

模板类分为：

```text
认知模板：身体、颜色、动作、生活物品、情绪、家庭成员。
故事模板：粘土娃娃晚安故事、生日成长书、刷牙睡觉、收玩具、第一次上幼儿园。
```

模板类不是从 0 编故事，而是在固定结构里填入孩子照片、孩子特点、玩偶、公仔、家庭场景和少量文案。

### 2. 原创类

适合有情节的绘本，例如成长故事、行为引导、情绪引导、英语启蒙故事、虚构冒险故事。

原创类必须先确认：

```text
故事谁提供
角色谁提供
风格谁决定
用途是什么
比例是否确认
```

确认前不直接生成完整故事。

## 基本流程

```text
路线分流：模板类 / 原创类
→ 模板或故事来源确认
→ 主角和素材来源确认
→ 风格确认
→ 用途、页数、比例确认
→ 绘本定位
→ 页面拆分
→ 文案对白
→ 角色设定
→ 风格设定
→ 每页提示词
→ 打印 / 发布检查
```

## 先分流，再追问

用户刚说「我想做绘本」时，最重要的不是问 5 个细节，而是先判断：

```text
他是想快速套模板做成品？
还是想原创一个有情节的故事？
```

因此第一问必须先让用户选择「模板类」或「原创类」。

## 模板类的价值

模板类更适合引流和成交，因为用户容易理解：

- 用自己孩子照片做一本身体认知书；
- 用孩子和玩偶做一本粘土风晚安故事；
- 用孩子日常动作做一本动作认知书；
- 用孩子喜欢的玩具做一本生活物品认知书。

模板类重点是清晰、好做、好打印，不追求复杂故事。

## 原创类的价值

原创类适合更个性化的家庭故事。

但原创类更容易跑偏，所以必须确认：

- 用户有没有故事；
- 角色是否已有；
- 风格是否指定；
- 是否需要 AI 帮忙构思；
- 是否要打印。

## 绘本不是说教

亲子绘本可以有教育目标，但不能像说明书。

更好的路径是：

```text
真实场景 → 一个小冲突 → 一个小尝试 → 情绪变化 → 温柔收束
```

低龄绘本尤其要短：一页一个动作、一种情绪、一个画面重点。

认知书不需要强行讲故事，重点是：

```text
孩子本人形象 + 认知词 + 简单句 + 清楚版式
```

## 图片和文字分开处理

默认建议：

- 图片生成时不要直接生成文字；
- 先生成无文字画面；
- 再用 Canva / PPT / Keynote / 飞书 / 公众号编辑器加文字。

原因：AI 生成中文字容易错误，且后期修改困难。

## 外部生图工具策略

本 skill 第一版只提供提示词，不绑定具体生图工具。

用户可把提示词复制到：

- 即梦；
- Midjourney；
- DALL·E / GPT image；
- 豆包；
- 可灵；
- 其他图像模型。

本 skill 只负责让提示词结构稳定、角色一致、风格一致、页面目标清晰。

## 照片素材要求

如果用户要用孩子本人照片，提醒准备：

1. 正脸照；
2. 表情自然；
3. 光线清楚；
4. 至少 3-5 张；
5. 如果做身体 / 动作认知，最好有对应动作或部位照片；
6. 如果做粘土娃娃晚安故事，最好有孩子照片 + 玩偶 / 公仔照片。

同时提醒保护孩子隐私，不要把原始照片公开上传到不信任的平台。

## 打印优先级

如果用户要打印，必须先提醒用户：比例和尺寸要向打印店 / 制作平台确认清楚。

没有确认比例前，只能输出策划草案，不生成最终生图提示词。

必须确认：

- 尺寸；
- 比例；
- 页数；
- 封面封底；
- 装订方式；
- 边距和出血；
- 图片分辨率；
- 文字是否后期排版。

打印问题不确认，后面图片比例很容易全部不合适。
